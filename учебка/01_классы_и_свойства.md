# C# — Классы и свойства

---

## 1. Что такое класс

Класс — это шаблон объекта. Как чертёж. По чертежу можно создать много объектов.

```csharp
// Определение класса
public class Cat
{
    // поля (данные)
    public string Name;
    public int Age;
}

// Создание объекта
Cat myCat = new Cat();
myCat.Name = "Мурзик";
myCat.Age = 3;
```

## 2. Свойства — это НЕ поля

В C# вместо публичных полей принято использовать **свойства** (Properties).

**Поле** — просто переменная внутри класса.  
**Свойство** — управляемый доступ к данным через `get` и `set`.

### Полная форма свойства:
```csharp
public class Product
{
    private decimal _price; // приватное поле (хранилище)

    public decimal Price    // публичное свойство
    {
        get { return _price; }         // когда читаем: product.Price
        set { _price = value; }        // когда пишем: product.Price = 100
    }
}
```

### Автосвойство (короткая форма — используется чаще всего):
```csharp
public class Product
{
    public decimal Price { get; set; }    // get и set автоматические
    public string Name { get; set; }
    public int Count { get; set; }
}
```
Это то же самое что выше, но C# сам создаёт скрытое поле.

### Свойство только для чтения:
```csharp
public string FullName { get; private set; }  // снаружи только читать, внутри можно писать
public int Total { get; }                      // только читать (задаётся в конструкторе)
```

## 3. Вычисляемые свойства — БЕЗ set

Свойство может вычислять значение на лету, не хранить ничего:

```csharp
public class Product
{
    public decimal Price { get; set; }
    public int Sale { get; set; }   // скидка в процентах

    // Это свойство ВЫЧИСЛЯЕТСЯ, а не хранится
    public decimal FinalPrice
    {
        get
        {
            if (Sale > 0)
                return Price * (1 - Sale / 100m);
            return Price;
        }
    }

    public bool HasDiscount
    {
        get { return Sale > 0; }
    }
}
```

### Ещё короче — через `=>` (expression body):
```csharp
public decimal FinalPrice => Sale > 0 ? Price * (1 - Sale / 100m) : Price;
public bool HasDiscount => Sale > 0;
public bool IsOutOfStock => Count == 0;
```

**Именно так написано в проекте ShoeShop** (файл `Entities/Product.cs`):
```csharp
public bool HasDiscount => Sale > 0;
public bool IsBigDiscount => Sale > 15;
public bool IsOutOfStock => Count == 0;
public decimal FinalPrice => HasDiscount ? Math.Round(Price * (1 - Sale / 100m), 2) : Price;
```

## 4. Конструктор

Конструктор — метод, который вызывается при создании объекта:

```csharp
public class User
{
    public string Name { get; set; }
    public string Email { get; set; }

    // Конструктор
    public User(string name, string email)
    {
        Name = name;
        Email = email;
    }
}

// Использование:
User u = new User("Иван", "ivan@mail.ru");
```

Если конструктор не написать — C# создаёт пустой автоматически.

## 5. Модификаторы доступа

| Слово | Кто видит |
|-------|-----------|
| `public` | все |
| `private` | только внутри класса |
| `protected` | класс и его наследники |
| `internal` | только внутри проекта |

В сущностях EF Core всё `public` — чтобы EF мог читать и писать данные.

## 6. Статические классы и поля

`static` — значит одно на всю программу, не привязано к объекту:

```csharp
public static class CurrentUser
{
    public static int Id { get; set; }
    public static string FullName { get; set; }
    public static string Role { get; set; }
}

// Использование (без new!):
CurrentUser.Id = 5;
CurrentUser.Role = "Administrator";
```

В проекте `CurrentUser.cs` — именно такой класс. Хранит данные залогиненного пользователя.

## 7. Навигационные свойства (для EF Core)

Это свойства, которые ссылаются на другие сущности:

```csharp
public class Product
{
    public int IdProd { get; set; }
    public string Descrip { get; set; }

    // Внешний ключ (просто int)
    public int CatId { get; set; }

    // Навигационное свойство — объект категории
    public virtual Category Cat { get; set; }
}
```

Когда EF подгружает продукт с `.Include(p => p.Cat)` — он автоматически заполняет `Cat` объектом из таблицы `categories`.

## Шпаргалка

```
поле           →  private int _age;
автосвойство   →  public int Age { get; set; }
только чтение  →  public int Age { get; }
вычисляемое    →  public bool IsAdult => Age >= 18;
статическое    →  public static string AppName { get; set; }
```
